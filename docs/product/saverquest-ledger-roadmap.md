# SaverQuest Ledger Roadmap

## Goal

SaverQuest를 `절약 계산기 데모`에서 `개인 가계부 + 절약 인사이트 앱`으로 전환한다.

## Product Direction

- 핵심 제품: 로컬 기반 개인 가계부
- 차별점: 기록 후 바로 예산/인사이트로 이어지는 절약 관점 UX
- 현재 구현 완료 범위:
  - 거래 기록 입력
  - 월 예산 저장
  - 홈 대시보드 실데이터화
  - 월간 리포트 실데이터화
  - 인사이트 실데이터화
  - 로컬 저장소 및 단위 테스트

## 50-Step Expansion Plan

### Phase 1: Foundation
1. 제품 포지셔닝을 `경량 가계부 + 절약 인사이트`로 확정
2. 거래 도메인 모델 정의
3. 카테고리 체계 정의
4. 거래 유형(지출/수입) 규칙 정의
5. 월 예산 상태 모델 정의
6. 로컬 저장 포맷 정의
7. 요약 계산 서비스 정의
8. 컨트롤러 책임 범위 분리
9. 홈 정보 구조 재설계
10. 기록 화면 정보 구조 재설계

### Phase 2: Core Ledger
11. 거래 입력 폼 구현
12. 월 예산 입력 폼 구현
13. 거래 저장 구현
14. 월 예산 저장 구현
15. 홈 대시보드 실데이터 연결
16. 최근 거래 리스트 구현
17. 월간 리포트 실데이터 연결
18. 카테고리별 합계 구현
19. 인사이트 화면 실데이터 연결
20. 빈 상태 UX 구현

### Phase 3: Usability
21. 거래 수정 기능 추가
22. 거래 삭제 기능 추가
23. 날짜별 필터 추가
24. 월별 이동 기능 추가
25. 카테고리 필터 추가
26. 검색 기능 추가
27. 메모 UX 보강
28. 빠른 입력 프리셋 추가
29. 금액 입력 UX 개선
30. 홈 CTA 최적화

### Phase 4: Reporting
31. 주간/월간 전환 추가
32. 일자별 지출 추이 추가
33. 카테고리 비중 차트 추가
34. 수입/지출 밸런스 차트 추가
35. 예산 소진 예측 추가
36. 고정비/변동비 분리 추가
37. 반복지출 탐지 추가
38. 월간 비교 리포트 추가
39. 주요 변화 카테고리 표시 추가
40. 리포트 공유 포맷 정의

### Phase 5: Insight Engine
41. 상위 지출 카테고리 추천 규칙 강화
42. 예산 초과 경고 규칙 추가
43. 절약 가능 항목 추천 규칙 추가
44. 수입 대비 소비 비율 인사이트 추가
45. 카테고리별 목표 제안 추가
46. 사용자 메모 기반 힌트 추가
47. 개인화 인사이트 문장 품질 개선
48. 알림/리마인더 전략 정의
49. 장기 보관/백업 전략 정의
50. 내부 테스트와 릴리즈 품질 게이트 정리

## Implementation Notes

- 매 단계는 `데이터 모델 -> 상태 관리 -> UI -> 테스트` 순서로 구현한다.
- 광고는 핵심 플로우를 막지 않는 배너 중심으로 유지한다.
- 리포트/인사이트는 기본 접근 가능 상태를 유지한다.
- 클라우드 동기화는 Phase 5 이후로 미룬다.
