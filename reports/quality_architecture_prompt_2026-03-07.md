# SaverQuest Quality Architecture Prompt

## Master Prompt
현재 Flutter 앱을 아래 기준으로 리팩터링해줘.

### 필수 조건
- 확장성과 유지보수성을 최우선으로 설계
- SOLID 원칙 준수
- 객체지향 설계 적용
- 테스트 가능한 구조로 작성
- 모든 함수에 목적 설명 주석 작성
- 에러 발생 가능 지점 사전 분석
- 보안 취약점 고려
- 성능 병목 가능성 분석

### 코드 품질 규칙
- 하드코딩 금지
- 매직넘버 금지
- 중복 로직 금지
- 환경변수 분리
- 의존성 분리
- 로깅 구조 포함

### 실행 순서
1. 현재 구조에서 하드코딩, 매직넘버, 결합도 높은 지점을 표로 정리
2. 환경 옵션, 로깅, 부트스트랩, 콘텐츠 저장소를 분리
3. 화면은 데이터 제공자/서비스만 소비하도록 변경
4. 변경된 함수마다 목적 주석 추가
5. 보안/성능/에러 포인트를 문서화
6. `flutter analyze`, `flutter test`, `flutter build apk --debug`로 검증

### 기대 산출물
- 수정 파일 목록
- 구조 개선 이유
- 보안 점검 항목
- 성능 점검 항목
- 테스트 결과
